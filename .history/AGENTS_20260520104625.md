# Repository Instructions

This workspace is a set of independent training projects. Always work inside the task directory that owns the change and read that task's README before editing.

## Shared Rules

- Treat each `task_*` folder as a separate project with its own `package.json` and tooling.
- Prefer the smallest change that fits the task; do not introduce cross-task abstractions.
- Link to existing docs instead of repeating them. Start with [README.md](README.md), [task_0/README.md](task_0/README.md), and [task_0/dashboard/README.md](task_0/dashboard/README.md).
- If a request is outside the stack used by the current task, adapt the guidance for that stack instead of forcing the default frontend pattern.

## Stack-Specific Guidance

### `task_0`

- Use TypeScript with Webpack.
- Keep the build and output conventions already used in `task_0`.
- Verify changes against the task's TypeScript entry point and bundled output.

### `task_0/dashboard`

- Use React with Vite.
- Follow the existing ESLint, Jest, and Testing Library setup in this folder.
- Keep components functional and align with the current Vite-based structure.

### `task_1`, `task_2`, `task_3`

- Use the Webpack + jQuery + Lodash stack already present in these tasks.
- Preserve the task-specific differences in loaders, asset handling, module splitting, and dev-server setup.
- Do not replace the curriculum's jQuery flow with React.

### `task_5`

- Use TypeScript with Webpack and the existing branded-type patterns.
- Keep type changes narrow and consistent with the current strictness level.

## Validation

- Run the script that belongs to the task you changed, not a workspace-wide command.
- Prefer the task's own build, start, lint, or test command when it exists.
